MediCrew — Full System Design

Swarm architecture · B+C patient UX · 5 layers · Both patient & doctor modes

🏗️ Layer Architecture

L1 TRIAGE AGENT — fast model, instant urgency + routes to relevant doctors
L2 7 SENIOR DOCTORS — all fire in parallel, each owns their specialty
L3 SUB-AGENT SWARM — each doctor spawns N hypothesis sub-agents (Promise.all)
L4 DEBATE ROUND — doctors read each other's swarm results, challenge or agree
L5 SYNTHESIS — orchestrator merges all confidence scores → final ranked output

🔧 Technical Changes (what gets built)

1

New swarm orchestrator

Replace linear LangGraph pipeline with parallel swarm: L1 triage → L2+L3 parallel doctor+sub-agent fan-out → L4 debate → L5 synthesis

2

Token-level streaming SSE

Each sub-agent streams tokens as it thinks. Patient sees words appearing in the live feed. No more waiting for full node completion.

3

Question-pause mechanism

Sub-agents can emit a "need_clarification" event mid-run. Orchestrator pauses that swarm branch, surfaces question to patient, resumes on answer.

4

B+C patient UI + doctor debug view

Patient sees orbs + live feed. Doctors see the full swarm confidence panel.

5

Fix all 14 agent-review issues

Parallel specialists, fail-safe triage, rate limiting, guardrails, observability — all fixed as part of the rebuild.
