## Integration Findings

### Critical Gap: consult/page.tsx Never Updated

**Root cause:** `src/app/consult/page.tsx` still renders `<SwarmChat />` which routes exclusively to `/api/swarm/start`. Phase 2 (plan 02-03) built the complete SSE streaming system — `streamConsultation`, `AgentOverlay`, `RoutingNotice`, `CareSummary`, compliance gates — but never updated this single file.

**Orphaned components:**
- `src/components/consult/AgentOverlay.tsx` — never imported
- `src/components/consult/CareSummary.tsx` — never imported in consult page
- `src/app/api/consult/route.ts` — never called from patient-facing UI

**Active path gaps:**
- `/api/swarm/start` has no `detectEmergency` call
- `/api/swarm/start` has no `checkConsent` call
- `/api/swarm/start` never fires `consultation/completed` Inngest event
- SwarmChat SSE events do not include agentName/specialty fields

**Impact:** 9 requirements are partial/unsatisfied as a result of this single file.

### Secondary Gap: DEMO_PATIENT_ID Module Constant

`src/app/onboarding/page.tsx` line 18-20: `DEMO_PATIENT_ID` is a module-level constant evaluated once, not reactive state. First-time users without `patientId` in localStorage will have `'demo-patient'` written to their consent record.
