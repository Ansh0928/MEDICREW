  <action>
Create GET /api/patient/care-plan endpoint:

1. Auth gate: x-patient-id header, 401 if missing

2. Query in parallel (Promise.all):
   a. Latest consultation: prisma.consultation.findFirst where patientId, orderBy createdAt desc, select { id, urgencyLevel, recommendation, createdAt }
   b. Next pending check-in: prisma.checkIn.findFirst where { patientId, status: "pending" }, orderBy scheduledFor asc, select { id, scheduledFor, status }
   c. Recent check-ins: prisma.checkIn.findMany where patientId, orderBy createdAt desc, take 5, select { id, status, response, respondedAt, createdAt }
   d. Care team status: prisma.careTeamStatus.findUnique where patientId, select { statuses, updatedAt }

3. Derive monitoring status:
   - "active" if there is a pending check-in OR a consultation within the last 7 days
   - "inactive" otherwise

4. Extract action items from latest consultation recommendation (JSON):
   - nextSteps array (strings) from recommendation
   - If recommendation is null, return empty array

5. Return JSON:

```json
{
  "monitoringStatus": "active" | "inactive",
  "nextCheckIn": { "scheduledFor": "ISO", "status": "pending" } | null,
  "latestConsultation": { "urgencyLevel": "...", "createdAt": "ISO" } | null,
  "actionItems": ["..."],
  "recentCheckIns": [{ "status": "...", "response": "...", "respondedAt": "..." }],
  "lastAgentActivity": { "agentName": "...", "message": "...", "updatedAt": "..." } | null
}
```
